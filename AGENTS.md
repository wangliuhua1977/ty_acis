# AGENTS.md

## 角色
你是本项目的实现型开发代理。目标是严格按项目基线推进“天翼视联 AI 智能巡检系统”，不要自由发散。

## 开发前先读
优先阅读：
- `docs/codex-context-min.md`

收到任务后，先判断所属模块：
- AI智能巡检
- 故障派单处理
- 报表中心
- 主题与术语
- 配置与基础能力

## 项目范围
只做三个核心模块：
1. AI智能巡检
2. 故障派单处理
3. 报表中心

固定闭环：
自动巡检发现问题 → 进入派单池 → 企业微信群通知 → 自动/人工确认恢复 → 报表沉淀

禁止把项目扩成大而全平台、通用工单平台、复杂多角色状态机。

## 技术冻结
- .NET 8 Windows 桌面
- 沿用现有分层、目录、命名与页面组织
- 地图：高德
- 本地轻量持久化
- 通知：企业微信群 webhook
- API 接入参数不进界面，统一本地配置读取

## 业务硬约束
### AI智能巡检
- 巡检组来自接口设备清单
- 巡检组独立配置首次时间、每日次数、间隔
- 同组串行，不并发抢跑
- 在线状态为基础必选项
- 播放检查必须支持“重试 + 协议切换重试”
- 画面异常支持接口判定 + 本地截图分析
- 地图必须是核心工作区，实时联动巡检过程
- 点位可点开视频预览
- 故障点位红灯闪烁
- 结果可配置为上墙复核或直接派单

### 故障派单处理
- 巡检组级别配置自动派单 / 人工派单
- 通知走企业微信群 webhook
- 同一点位同一故障重复出现时合并，不重复新建
- 工单状态只保留：待派单 / 已派单
- 恢复状态只保留：未恢复 / 已恢复
- 恢复后自动发送恢复通知
- 管理员可调整处理单位、维护人、负责人及手机号

### 报表
首版只做：
- 巡检执行报表
- 故障统计报表
- 派单处置报表
- 责任归属报表
- 重点未恢复清单
